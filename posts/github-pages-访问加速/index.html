GitHub Pages 在国内本身就慢、不稳定，想让别人也能稳定快速访问，最靠谱、成本最低的方案是：套国内 CDN / 反代加速。

一、最简单：用免费国内 CDN 加速（推荐）

原理：把你的 GitHub Pages 放到国内节点缓存，用户访问 CDN 节点

1. 用 Cloudflare + 国内中转（最通用）

适合：你有自己的域名步骤：

  1. 域名 DNS 改成 Cloudflare

  2. 开启橙色云朵（代理）

  3. 开启：

    • Auto Minify
    • Brotli
    • Cache Level: Standard

  4. 页面规则设置缓存静态资源

效果：国外很快，国内会明显改善，但偶尔仍抽风。

二、国内最稳：使用「国内反代服务」

用免费公益反代（别人直接用）

网上有很多公益反代 GitHub Pages 的域名，别人直接访问反代地址即可：

  • ghproxy.com

  • gh.ddlc.top

  • fastly.jsdelivr.net 等

用法：

原地址：https://xxx.github.io/yyy

反代后：https://ghproxy.com/https://xxx.github.io/yyy

优点：

  • 不用你做任何配置

  • 别人直接访问加速链接

  • 完全免费

缺点：

  • 依赖第三方服务，可能不稳定

  • 不适合长期正式项目

三、自建反代 / 镜像（最稳、可控）

如果你希望长期稳定、自己掌控，可以自建反向代理。

方案 A：用 云服务器 + Nginx 反代

  1. 买一台轻量服务器（阿里云 / 腾讯云 / 华为云 学生机很便宜）

  2. Nginx 配置反向代理到 GitHub Pages

  3. 绑定你自己的域名，解析到国内服务器

示例 Nginx 配置：

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass https://xxx.github.io;
        proxy_set_header Host xxx.github.io;
        proxy_set_header User-Agent $http_user_agent;
    }
}

优点：

  • 国内访问飞快

  • 域名干净

  • 别人访问你的域名即可

方案 B：使用 Vercel / Netlify 替代 GitHub Pages

这两个平台国外节点好，国内访问也比 GitHub 快很多。

操作：

  1. 把仓库导入 Vercel / Netlify

  2. 自动构建部署

  3. 绑定域名

优点：

  • 比 GitHub Pages 快很多

  • 免费额度足够个人 / 小团队使用

  • 支持 https、自动部署

四、纯静态站：直接托管到国内平台

如果只是静态页面（HTML/CSS/JS），可以直接搬到国内托管：

  • Gitee Pages（码云）

  • 阿里云 OSS + 静态网站

  • 腾讯云 COS + CDN

速度：国内几乎秒开

你该选哪个？

  • 不想折腾、只想给别人快一点 → 用公益反代链接

  • 有自己域名，想要稳定 → Cloudflare + 自己域名

  • 正式项目、要长期用 → 自建 Nginx 反代 或 迁移到 Vercel / 国内对象存储

  • 完全不想碰服务器 → 直接改用 Gitee Pages
